## Plan: Mesh → OBJ Converter for DownloadAssets

### Problem
Downloaded `.mesh` files can't be drag-dropped into Roblox Studio — `.mesh` is Roblox's internal binary format, not a standard 3D format. Users get files they can't open.

### Solution
Parse the Roblox `.mesh` format (v1–v5) during download and save as `.obj` (which Studio CAN import) instead of `.mesh`. If parsing fails, fall back to saving the raw `.mesh`.

### Implementation

#### 1. Add `mesh_to_obj(body)` function (before `download_assets`, ~line 2649)

A standalone pure function: input = mesh body string, output = OBJ string or nil.

**Version detection:** Check first 8 bytes for `"version "`, read version string up to `\n`.

**v1.00 / v1.01 (ASCII text):**
- Parse face count from line 2
- Split the remaining data by `"]["` to extract bracketed `[x,y,z]` tuples
- Group into vertices (3 tuples each: position, normal, UV)
- v1.00: scale position by 0.5, flip UV V (`1.0 - tv`)
- v1.01: use as-is

**v2.00–v5.01 (binary, little-endian):**
- Find first `\n`, read binary from there
- Read header (12/16/24/32 bytes depending on version) → get `numVerts`, `numFaces`, `vertexStride`
- Read vertices: 40 bytes each (or 36 if stride=36):
  - Position: `float32[3]` (bytes 0–11)
  - Normal: `float32[3]` (bytes 12–23)
  - UV: `float32[2]` (bytes 24–31)
  - Tangent: `int8[4]` (bytes 32–35) — skipped
  - Color: `uint8[4]` (bytes 36–39) — skipped (only if stride=40)
- Read faces: 12 bytes each (`uint32[3]`)
- Skip envelopes, LODs, bones, subsets, FACS (not needed for OBJ)

**Binary float reading (no `string.unpack` dependency):**
```lua
local function read_le_u32(body, offset)
    local b1, b2, b3, b4 = string.byte(body, offset + 1, offset + 4)
    return b1 + b2 * 256 + b3 * 65536 + b4 * 16777216
end
local function read_le_f32(body, offset)
    local u = read_le_u32(body, offset)
    -- IEEE 754 reconstruction from uint32
    ...
end
```
Uses `string.byte` + manual IEEE 754 bit math — works on ALL executors, no `buffer` API dependency.

**OBJ output format (matches existing `export_map_obj` style):**
```
# Converted from Roblox mesh (version X.XX)
v x y z
v x y z
...
vt u v
vt u v
...
vn nx ny nz
vn nx ny nz
...
f v1/vt1/vn1 v2/vt2/vn2 v3/vt3/vn3
...
```
Uses table buffer + `table.concat` for efficiency (same pattern as `export_map_obj`).

#### 2. Modify `download_one()` (line ~2869)

After `detect_ext` returns `"mesh"`:
```lua
if ext == "mesh" and O.DownloadAssetsConvertMesh ~= false then
    local obj_data = mesh_to_obj(body)
    if obj_data then
        ext = "obj"
        body = obj_data
        mesh_converted += 1
    else
        mesh_convert_fail += 1
        -- ext stays "mesh", raw body saved as fallback
    end
end
```

### Risk assessment
- **Low risk:** `mesh_to_obj` is a pure function wrapped in pcall — if it errors, the fallback saves the raw `.mesh` file (current behavior)
- **No new dependencies:** Uses only `string.byte`, `string.sub`, `string.format`, `table.concat` — all standard Luau
- **Backward compatible:** `DownloadAssetsConvertMesh` defaults to `true`; users who want raw `.mesh` can set it to `false`
